1. 网络基础
    1) IP地址
        每一台联网的电脑都会有一个地址, 用于和其他计算机进行通讯.
        IP地址主要有两个版本, IPV4和IPV6版本.

        IPV4版本的地址格式是由四个十进制数字组成，每个数字的取值范围是0到255，数字之间用点（.）分隔.
        共有255**4个ip地址, 不是很够用.
        如
            192.168.0.1

        IPV6版本的地址格式是是128位的，通常表示为8组16位的十六进制数，每组之间用冒号（:）分隔.
        共有2**128个ip地址, 够用了.
        如
            2001:0db8:85a3:0000:0000:8a2e:0370:7334

        ifconfig 命令, 查看本机的ip地址, 若无法使用, 需要通过软件包管理器安装.

    2) 主机名
        每一台电脑除了对外联络地址(IP地址)外, 也可以有一个自己的名字, 称为主机名.

        hostname 命令, 查看主机名

        修改主机名
            hostname set-hostname 新的主机名

    3) 域名解析
        IP地址对人不友好, 不易记忆, 便出现了域名, 当想要访问某个地址的时候, 可以通过一串好记忆的字符串, 经解析, 连接到对应IP地址.

        如
            浏览www.example.com, 若没有缓存其IP地址, 浏览器将会向公开的DNS服务器问询其地址, 若存在, 将该地址给予浏览器用于访问.

        Linux中IP地址缓存文件 /etc/hosts
        Windows中IP地址缓存文件 C:\Windows\System32\drivers\etc\hosts

    4) DHCP
        动态获取IP地址, 即每次重新启动设备后都会动态的分到一个IP地址

        如果需要固定IP, 可修改 /etc/netplan/01-netcfg.yaml 文件
        在使用 systemctl restart network 重启网络服务
        注意
            对于不同的Liunx发行版, IP地址所保存的文件不同.


2. 网络传输

    1) ping 检查指定网络服务器是否可联通
        语法: ping 选项 ip或主机名
        选项
            -c 次数 : 指定检查的次数, 不指定-c选项, 将会一直执行 (Windows中默认有次数限制)
        示例
            域名www.example.com是否可以连通
                ping -c 4 www.example.com

    2) wget 非交互式文件下载器
        wget命令可以再命令行内下载网络文件
        语法: wget 选项 网络文件地址
        选项
            -b : 后台下载, 会将日志写入当前工作目录的wget-log文件, 通过tail命令可以监控后台下载进度, tail -f wget-log

        注意
            无论下载是否完成, 都会生成要下载的文件, 如果下载未完成, 需及时清理不可用文件

    3) curl 数据传输工具
        curl 是一个在命令行或脚本中向/从服务器传输数据的工具，支持许多协议，如 HTTP、HTTPS、FTP、FTPS、SCP 等.

        语法: curl 选项 网络地址
        选项
            -O : 下载文件
            -X : 指定请求方法
            -d : 发送数据

        示例
            发送 GET 请求
                curl http://example.com
            发送带有头部的请求：
                curl -H "Content-Type: application/json" -X POST -d '{"key1":"value1", "key2":"value2"}' http://example.com/resource
            显示响应头信息
                curl -I http://example.com

        
3. 端口
    端口是设备与外界通讯交流的出入口.
    端口可分为两类
        物理端口 : 又可称之为接口, 是可见的端口, 如USB接口, RJ4网口, HDMI端口等
        虚拟端口 : 是指计算机内部的端口, 是不可见的, 是用来操作系统和外部进行交互使用的

    通常一个操作系统支持2**16-1, 即65535个端口, 这6万个端口分为三类使用
        1) 公认端口 : 1~1023, 通常用于一些系统内置或知名程序的预留使用, 如ssh服务的22端口, Https服务的443端口, 非特殊需要, 切勿占用这些端口
        2) 注册端口 : 1024~49151, 通常可以随意使用, 用于松散的绑定一些程序或服务
        3) 动态端口 : 49152~65535, 通常不会固定绑定程序, 而是当程序对外进行网络链接时, 用于临时使用

    可以理解为IP地址是电脑在网络中的小区地址, 端口是小区中具体某个程序住的房间号, 协议是以什么交通工具到达该位置.

    1) nmap 查看端口占用
        nmap命令
        语法: nmap 被查看的IP地址
        示例
            查看本机的端口占用情况
            nmap 127.0.0.1

    2) netstat 查看指定端口的占用情况
        netstat命令
        语法: netstat -anp | grep 指定端口号

    
4. 进程
    程序运行在操作系统中, 是被操作系统所管理的.
    为管理运行的程序, 每一个程序在运行的时候, 便被操作系统注册为系统中的一个 进程

    并会为每一进程都分配一个独有的 进程ID, 即进程号

    1) ps 查看进程
        ps命令
        语法: ps 选项
        选项
            -e : 显示出全部的进程
            -f : 以完全格式化的形式展示信息(展示全部信息)
        一般来说, 固定的用法就是 ps -ef , 即列出全部进程信息
        
        进程信息的表头
            UID : 进程所属的用户ID
            PID : 进程的进程号ID
            PPID : 进程的父ID(即启动此进程的其他进程)
            C : 此进程的CPU占用率(百分比形式)
            STIME : 进程的启动时间
            TTY : 启动此进程的终端序号, 如显示?, 表示非终端启动
            TIME : 进程占用CPU的时间
            CMD : 进程对应的名称或启动路径或启动命令

    2) kill 关闭进程
        Windows系统中, 可以通过图像化的任务管理器关闭进程.

        Linux系统中, 可以通过kill命令关闭进程
        语法: kill 选项 进程ID
        选项
            -9 : 表示强制关闭进程, 不使用此选项会向进程发送信号要求其关闭, 但是否关闭要看进程自身的处理机制
        
        示例
            关闭more进程
                ps -ef | grep more
                kill -9 进程ID
